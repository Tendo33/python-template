# Release

## Current Truth

The repository uses `.github/workflows/release.yml` for tag-triggered releases.

The release path is:

1. Update version number.
2. Run local verification.
3. Build package artifacts and confirm generated frontend dependency/cache
   directories are not included in the sdist.
4. Create and push a `v*` tag.
5. GitHub Actions generate or update the GitHub Release.

## Release Notes

Release notes are generated by `scripts/generate_release_notes.py`.

The script reads:

- target tag
- previous tag
- changelog
- commits

With model credentials it attempts an AI-generated summary. If that fails or no
credentials exist, it falls back to deterministic highlights.
