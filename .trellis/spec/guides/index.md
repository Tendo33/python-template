# Guides

Use these guides to avoid design mistakes before writing code.

| Guide | Purpose | When to Use |
| --- | --- | --- |
| [Task Flow](./task-flow.md) | Default workflow | Any task |
| [Pre-Implementation Checklist](./pre-implementation-checklist.md) | Readiness questions before code | Non-trivial changes |
| [Cross-Layer Thinking Guide](./cross-layer-thinking-guide.md) | Python/backend/frontend contract thinking | Changes crossing frontend/backend/static serving |
| [Review Checklist](./review-checklist.md) | Final handoff check | Before completion |

## Core Habit

Search before adding a new helper, component, model, config value, or endpoint.
Most avoidable complexity starts when a second copy of a pattern is created
without noticing the first one.
