# Task Flow

Use this flow for non-trivial work.

1. Read the project entry docs.
2. Identify whether the work is backend, frontend, full stack, scripts, or docs.
3. Read the matching current docs before standards docs.
4. Make the smallest change that fully solves the request.
5. Update docs only when behavior, structure, commands, scripts, or public APIs
   changed.
6. Run the verification scope from `shared/verification.md` or the project's
   local verification reference.
7. Summarize what changed and what was verified.

## Pushback Triggers

Pause and clarify when:

- The request conflicts with current project facts.
- The task asks for a future layer to be treated as existing behavior.
- The smaller correct fix is materially different from the requested approach.
- Verification cannot prove the requested outcome.
